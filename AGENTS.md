# AGENTS.md — AI Agent Code Modernizer

This file describes the AI agents available in this repository for coding assistants (GitHub Copilot, Claude, Gemini, and others).

## Repository Purpose

This repository contains the **AI Agent Code Modernizer** — an MCP-exposed AI agent that analyzes code written with **Semantic Kernel** or **AutoGen** and modernizes it to **Microsoft Agent Framework (MAF)**.

## Project Structure

| Directory | Purpose |
|-----------|---------|
| `AIAppsModernization/` | Core modernizer agent — MCP server, tools, and agent logic |
| `SemanticKernelSamples/` | Sample AI applications in Semantic Kernel and AutoGen (test inputs) |
| `infrastructure/` | ARM templates, Docker, deployment configuration for Azure Container Apps |
| `docs/` | Architecture diagrams and documentation |
| `specs/` | Modernization specifications generated by the ModernMint agents |
| `plans/` | Implementation plans generated by the ModernMint planner agent |
| `.github/agents/` | GitHub Copilot agent definitions |

## Available Agents

### @modernmint-spec

**Purpose**: Creates modernization specifications from natural-language requirements.

**What it does**:
- Scans the workspace to identify Semantic Kernel and AutoGen patterns in existing code
- Catalogs all capabilities the current application provides
- Produces a single, comprehensive specification at `specs/modernization-spec.md`
- Generates a quality checklist at `specs/checklists/spec-quality.md`
- Focuses exclusively on **WHAT** needs to change — never prescribes **HOW**

**When to use**: When you need to define the scope and requirements of an AI application modernization before any code changes begin.

**Output**: A technology-agnostic specification document with functional requirements, user scenarios, success criteria, and a quality checklist.

**Next step after completion**: Use `@modernmint-planner` to create the technical implementation plan.

### @modernmint-planner

**Purpose**: Generates technical implementation plans from modernization specifications.

**What it does**:
- Reads the modernization specification from `specs/modernization-spec.md` (if available)
- Scans the workspace to build a complete technical inventory of Semantic Kernel and AutoGen patterns
- Maps every detected pattern to its Microsoft Agent Framework equivalent using official migration guides
- Produces a phased, step-by-step implementation plan under `plans/`
- Generates a quality checklist at `plans/checklists/plan-quality.md`
- Focuses exclusively on **HOW** to modernize — concrete code transformations, dependency changes, and verification steps

**When to use**: After `@modernmint-spec` has produced a specification (or directly if you want a plan based on codebase analysis alone).

**Output**: One or more plan documents under `plans/` with file-by-file transformation tasks, before/after code patterns, and verification criteria.

**References**: Grounded in the official [SK → Agent Framework migration guide](https://learn.microsoft.com/en-us/agent-framework/migration-guide/from-semantic-kernel/) and [AutoGen → Agent Framework migration guide](https://learn.microsoft.com/en-us/agent-framework/migration-guide/from-autogen/).

## ModernMint Workflow

1. **Specify** → `@modernmint-spec` analyzes the codebase and user requirements to produce a modernization specification
2. **Plan** → `@modernmint-planner` reads the specification and codebase, then generates a phased implementation plan with file-by-file transformation tasks
3. **Implement** → Follow the plan phases in order to execute the modernization

## Key Conventions

- **Python 3.10+** with async/await patterns throughout
- **Type annotations** using `Annotated` for all tool parameters
- **Environment configuration** via `.env` files and `python-dotenv`
- **Azure AI Foundry** for model hosting
- **MCP** (Model Context Protocol) for tool exposure

## Frameworks Involved

### Source Frameworks (to modernize FROM)

- **Semantic Kernel**: `Kernel`, `@kernel_function`, `ChatHistory`, `FunctionChoiceBehavior`, plugins, planners, connectors, memory stores
- **AutoGen**: `AssistantAgent`, `UserProxyAgent`, `RoundRobinGroupChat`, `FunctionTool`, `TextMentionTermination`, multi-agent orchestration

### Target Framework (to modernize TO)

- **Microsoft Agent Framework**: `AzureAIClient`, `create_agent()`, `WorkflowBuilder`, standard Python tool functions, thread persistence, streaming, MCP server support
